DatabaseInput Node sample

The DatabaseInput Node sample shows how to receive and process database updates by using an integration node message flow.

NOTE: In WebSphere Message Broker Version 8.0, IBM Integration Bus 9.0 and later, message model schema files contained in libraries are the preferred way to model messages for most data formats, including the new DFDL domain. Message sets continue to be supported, and are required if you use the MRM or IDOC domains. You can continue to import and deploy message sets for use in message flows. However, if you need to create one or more message sets, message definitions, or message categories when using this sample, you must first enable message set development in the IBM Intgration Toolkit. For more information, see Enabling message set development in the IBM Integration Bus documentation.

Set up the database

You can set up the sample in one of the following ways:

  • Import and partially deploy the sample

    This option imports the sample files into your workspace. This option also creates an integration server and WebSphere MQ queues. Use this option if you are not familiar with setting up database tables. An SQL script to set up the tables is imported into your workspace; see Setting up the database tables by using SQL scripts. Run this script before deploying the broker archive (BAR) file to the integration server.

  • Import the sample

    This option imports the sample files into your workspace. Additional manual steps are required to configure the sample for testing

You can import or import and deploy a sample only when you use the information center that is integrated with the IBM Integration Toolkit.
